\chapter{Experiments and Results}\label{chap:er}

\section{Definition of Correct Identification}\label{sec:identity_definition}
A known challenge in evaluating \ac{TCG} recognition systems is the handling of reprints. As previously illustrated in \Cref{fig:tiny_details}, a single mechanically unique card is frequently reprinted across multiple editions. These reprints often utilize the same primary artwork, differing only in localized details such as a set symbol, a holographic stamp, or copyright text.

Because the proposed architecture (\Cref{sec:model_architecture}) processes the entire card as a holistic visual fingerprint, the network inherently prioritizes macro-level features (e.g., silhouettes and/or color palettes) over micro-level typographical differences.

Thus, this thesis defines a \ii{True Positive (Correct Match)} primarily at the \ii{Name-Level}. If the system retrieves an embedding that matches the canonical name of the queried card, the prediction is considered correct, regardless of the specific edition. While exact-edition matching is evaluated to demonstrate the system's operational limits, achieving robust edition verification is better suited for a secondary pipeline, which is designated as future work.

\section{Evaluation Metrics}
Because the recognition pipeline utilizes Open-Set Metric Learning rather than standard closed-set classification, traditional metrics like simple accuracy are insufficient for evaluating the structural robustness of the model. The system operates as an image retrieval mechanism, comparing the extracted feature vector of a queried card against a known database using cosine similarity.

\subsection{Verification Performance (\acs{FMR} \& \acs{TMR})}
To evaluate the discriminative power of the learned embedding space, metrics adapted from biometric verification systems are utilized~\cite{jain2011biometrics}. Specifically, the performance is quantified using the \acf{FMR} at a fixed \acf{TMR} of 95\,\%.

Let $s(\mathbf{x}_i, \mathbf{x}_j)$ denote the cosine similarity between the feature embeddings $\mathbf{x}_i$ and $\mathbf{x}_j$ extracted from two images. We define two sets of evaluation pairs:
\begin{itemize}
    \item \bb{Genuine Pairs} ($\mathcal{P}_{\text{gen}}$): A tuple $(\mathbf{x}_i, \mathbf{x}_j)$ where both embeddings represent the same card. In synthetic validation, this pairs an augmented query with its clean digital template. In real-world testing, this pairs a physical camera photograph with its corresponding clean digital template.
    \item \bb{Impostor Pairs} ($\mathcal{P}_{\text{imp}}$): A tuple $(\mathbf{x}_i, \mathbf{x}_j)$ where the query image and the reference template belong to distinct cards.
\end{itemize}

The \ac{TMR} defines the proportion of genuine pairs correctly matched with a similarity score equal to or above a given decision threshold $\tau$. Conversely, the \ac{FMR} defines the proportion of impostor pairs that falsely yield a similarity score equal to or above $\tau$.

To compute the \ac{FMR} @ 95\,\% \ac{TMR}, the operational threshold $\tau_{95}$ is empirically determined such that 95\,\% of all genuine pairs are accepted:
\begin{equation}
    \text{\ac{TMR}}(\tau_{95}) = \frac{|\{ (\mathbf{x}_i, \mathbf{x}_j) \in \mathcal{P}_{\text{gen}} \mid s(\mathbf{x}_i, \mathbf{x}_j) \geq \tau_{95} \}|}{|\mathcal{P}_{\text{gen}}|} = 0.95
\end{equation}

Once $\tau_{95}$ is established, the \ac{FMR} is calculated as:
\begin{equation}
    \text{\ac{FMR}} = \frac{|\{ (\mathbf{x}_i, \mathbf{x}_j) \in \mathcal{P}_{\text{imp}} \mid s(\mathbf{x}_i, \mathbf{x}_j) \geq \tau_{95} \}|}{|\mathcal{P}_{\text{imp}}|}
\end{equation}

This metric reflects the system's precision. An \ac{FMR} approaching zero at this operating point indicates that the model can successfully recognize 95\,\% of targeted cards despite noise and already discussed domain shift (\Cref{sec:domain_shift}), without confusing them with visually similar cards.

\subsection{Retrieval Performance (Top-\texorpdfstring{$k$}{k} Accuracy)}
While the \ac{FMR} provides a measure of the embedding space's structural integrity, the objective of the robotic sorter is the identification of physical cards under real-world conditions. This is a 1-to-$N$ retrieval problem.

To evaluate operational viability, performance is quantified using Top-$k$ Accuracy. For a given query, the system retrieves the $k$ database entries with the highest cosine similarity. If the ground-truth identity exists within this subset, the prediction is correct. 

\begin{itemize}
    \item \bb{Top-1 Accuracy}: Represents the zero-intervention success rate of the identification mechanism.
    \item \bb{Top-3 and Top-5 Accuracy}: Serve as metrics for potential human-in-the-loop fallback systems. High Top-5 accuracy indicates that when the network is uncertain (e.g., due to lighting conditions), the correct card remains clustered near the query in the embedding space.
\end{itemize}

\section{Experimental Setup}
To validate the model, performance was measured across two evaluation sets:
\begin{enumerate}
    \item \ii{Synthetic Validation Set}: Consists of $1,000$ digital templates isolated from the training dataiiThese images are subjected to the stochastic augmentation pipeline (defined in \Cref{tab:augmentations}) and matched against clean digital templates. This set tracks the integrity of the embedding space during training.
    \item \ii{Real-World Evaluation Set}: A manually curated dataset of \todo{insert number} unconstrained phyiiphotographs of \ac{MTG} cards. These images feature diverse backgrounds, card sleeves and perspective shifts to test the system's robustness against the domain shift discussed in \Cref{sec:domain_shift}.
\end{enumerate}

The network was trained for 60 epochs across 4 GPUs utilizing the distributed \ac{DDP} architecture detailed in \Cref{chap:sw}. The hardware processed a local batch size of 64 images per GPU, yielding an effective global batch size of 256. 

The embedding space was regularized via the ArcFace head with a margin $m=0.50$ and scale $s=64.0$. Optimization was performed using \ac{SGD} with a momentum of $0.9$ and a weight decay of $5 \times 10^{-4}$. The base learning rate parameter was defined as $\eta_{\text{base}} = 0.1$. 

To prevent catastrophic forgetting of the low-level visual features pre-trained on ImageNet, differential learning rates were applied: the \ac{ResNet}-50 backbone was optimized at a learning rate an order of magnitude smaller ($\eta = 0.1 \eta_{\text{base}}$) than the randomly initialized embedding and ArcFace heads ($\eta = \eta_{\text{base}}$). Furthermore, following the optimization strategy from the official ArcFace implementation~\cite{deng2022arcface}, a Polynomial Warmup learning rate scheduler was employed. The learning rate was linearly scaled up over the first two epochs to stabilize initial gradient updates, followed by a polynomial decay ($p=2.0$) over the remaining training steps.

\subsection{Embedding Space Integrity (\acs{FMR} \& Margins)}\label{subsec:exp_metric_learning}
\Cref{fig:training_metric_space} visualizes how the ArcFace objective function manipulates the feature vectors on the 512-dimensional hypersphere over the course of training.

\begin{figure}[H]
    \centering
    \input{./fig/tikz/training-metric-space.tex}
    \caption{Top: The progression of positive (genuine) and negative (impostor) cosine similarities. The shaded regions denote the standard deviation. Bottom: The corresponding logarithmic decay of the \ac{FMR}.}
    \label{fig:training_metric_space}
\end{figure}

The \ii{Cosine Similarities} plot (\Cref{fig:training_metric_space}, Top) illustrates the average geometric distance between pairs of images. In the initial epochs, the network struggles to separate the classes; the positive and negative similarities overlap, and the standard deviation bands are wide. As training progresses, the additive angular margin (introduced in \Cref{sec:arcface}) forces a distinct structural behavior:
\begin{enumerate}
    \item \ii{Inter-Class Separability:} The average similarity for impostor pairs is pushed toward $0.0$ (orthogonal vectors), ensuring different cards are separated.
    \item \ii{Intra-Class Compactness:} The average similarity for genuine pairs rises. The standard deviation band for positive pairs narrows over time, indicating that the network learns to map distorted queries of the same card into a compact cluster.
\end{enumerate}

This expanding margin causes the logarithmic decay observed in the \ac{FMR} plot (\Cref{fig:training_metric_space}, Bottom). By establishing a decision threshold $\tau$ (represented by the dashed line), the network minimizes the probability of misclassifying an impostor pair.

\subsection{Real-World Retrieval Performance}\label{subsec:exp_real_world}
To evaluate operational viability, the trained model extracted feature vectors from the Real-World Evaluation Set. These queries were compared against a pre-computed index database containing reference embeddings. The retrieval results are summarized in \Cref{fig:acc_barchart}. 

\begin{figure}[H]
    \centering
    \input{./fig/tikz/barplot-match-real.tex}
    \caption{Top-$k$ retrieval accuracy on unconstrained physical photographs.}
    \label{fig:acc_barchart}
\end{figure}

The system achieves a Top-1 Name Match accuracy of 97.8\,\%. This confirms that feature representations learned from synthetic data can generalize to physical assets. Furthermore, the Top-5 accuracy indicates that when the network encounters ambiguity, it ranks the correct identity within the top candidates. 

The data supports the premise proposed in \Cref{sec:identity_definition}: while Name Matching is accurate, Exact Edition matching drops to 83.7\,\%. This highlights the limitations of using a global visual feature extractor for fine-grained typography.

\subsubsection{Confidence Distribution}
To analyze the system's decision-making process, \Cref{fig:sim_histogram} visualizes the distribution of the Top-1 cosine similarity scores.

\begin{figure}[H]
    \centering
    \input{./fig/tikz/distribution-cosSim.tex}

    \label{fig:sim_histogram}

The histogram indicates a distinct operational separation. Correctly identified cards yield similarity scores primarily between $0.60$ and $0.85$. In contrast, incorrect matches cluster around the $0.40$--$0.50$ range. In an automated environment, this distribution implies that a confidence threshold (e.g., $\tau = 0.55$) can be established. Predictions falling below this threshold can be routed for manual review, reducing the rate of silent failures during sorting.
