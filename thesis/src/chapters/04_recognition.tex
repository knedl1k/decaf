\chapter{Card Recognition Pipeline}\label{chap:rec}

\section{Data Acquisition}
As was previously mentioned in \refsec{sec:scarcity}, one of the primary challenges in training deep learning models for \ac{TCG} identification is \ii{data scarcity}. While the total number of unique cards in circulation is vast (often exceeding tens of thousands for games like \ac{MTG}), labeled datasets containing real-world photographs of these cards are virtually non-existent. Furthermore, collecting and manually labeling a sufficiently large dataset of physical cards is prohibitively expensive and time-consuming.

To address this limitation, this thesis adopts a \ii{synthetic data generation} approach. High-quality reference images, serving as ground truth templates, were retrieved via the Scryfall Bulk Data API~\cite{scryfall}. This service provides a comprehensive catalogue of all printed cards in a structured JSON format, enabling the automated generation of training samples.

\subsection{Scraping Pipeline and Indexing}
A custom data collection script, \texttt{data\_scraper.py}, was developed to process metadata and download high-resolution scans. A simple file indexing strategy was also designed, aimed at achieving a balance between machine uniqueness and human readability, a feature essential for manual inspection and debugging.

Instead of relying solely on human-unreadable identifiers (such as \acp{UUID}), a hybrid naming convetion derived from the card's canonical \ac{URI} is proposed. Each file name combines the card edition, collector number, and name with a \ac{UUID}: 
\begin{equation}
    \texttt{filename} = \texttt{edition\_collectorNumber\_name-UUID.png}
\end{equation}
For example, a card from the \ii{Zendikar} set is stored as:
\begin{center}
    \texttt{zen\_21\_kor-outfitter-00006596-1166-4a79-8443-ca9f82e6db4e.png}
\end{center}
This format allows for immediate visual identification of the card's edition and variant without requiring a lookup table, while the \ac{UUID} suffix guarantees collision-free storage even for edge cases (e.g., duplicate names across sets).

\subsection{Handling Complex Layouts}
The script also implements logic to handle complex card layouts, such as \acp{DFC} (e.g., transform and modal \acp{DFC}). These entries, identified by the \texttt{layout} property in the JSON metadata, contain multiple faces within a single object. The pipeline detects these instances and automatically splits them into separate template images, appending \texttt{\_front} or \texttt{\_back} suffixes to their identifiers. This ensures the network learns to independently classify either face of the physical card without conflating the features of both sides.

\section{Domain Shift}\label{sec:domain_shift}
When a neural network is trained exclusively on clean digital templates but deployed to recognize physical cards captured by a camera, it typically experiences a severe degradation in predictive accuracy; a phenomenon documented as \ii{domain shift}~\cite{quinonero2009dataset}. The challenge of adapting models from synthetic source domains to real-world target distributions is a well-recognized bottleneck, particularly in comparable fine-grained visual classification tasks~\cite{smith2021leveraging}.

The reference images are characterized by uniform lighting, precise orthogonal alignment, the absence of lens distortion, and flawless surfaces. In contrast, the target domain consists of physical photographs taken using industrial sensors or consumer webcams. These images exhibit a high degree of variability, including significant lens distortion, uneven ambient lighting, sharp shadows, reflections from holographic foils, physical wear and tear, and cluttered backgrounds (e.g., textured tables or playmats).

\section{Synthetic Data Generation}
To bridge the domain shift described above, the training pipeline incorporates an on-the-fly synthesis module designed to simulate the artifacts inherent to physical image acquisition.

\subsection{Removing Sharp Edges}
Digital templates are often stored as rectangular images with transparent edges encoded via an alpha channel. The alpha channel $\bm{\alpha}$ is explicitly extracted and the linear interpolation is performed between the card's RGB tensor $\mathbf{X}_{\text{card}}$ and the simulated noisy canvas $\mathbf{X}_{\text{cv}}$:
\begin{equation}
    \mathbf{X}_{\text{composite}} = \mathbf{X}_{\text{card}} \odot \bm{\alpha} + \mathbf{X}_{\text{cv}} \odot \left(\mathbf{1} - \bm{\alpha}\right)
\end{equation}
where $\odot$ denotes the component-wise\footnote{also known as Hadamard} product. This ensures that the synthetic images mimic the continuous edge blending observed when a physical card is placed on a textured playing mat, forcing the network to focus on the internal visual features, as demonstrated in \reflst{lst:alpha_blend}.
\begin{lstlisting}[language=Python, caption={Alpha compositing for synthetic background generation.}, label={lst:alpha_blend}]
new_w, new_h = int(w * scale), int(h * scale)
resized_card = cv2.resize(image, (new_w, new_h), interpolation=cv2.INTER_AREA)
x_offset, y_offset = (target_size - new_w) // 2, (target_size - new_h) // 2

# Extract RGB and normalized Alpha channels
bgr = resized_card[:, :, :3]
alpha = resized_card[:, :, 3] / 255.0
alpha_3d = np.expand_dims(alpha, axis=2)

# Extract Region of Interest (ROI) from the simulated background
roi = canvas[y_offset : y_offset + new_h, x_offset : x_offset + new_w]

# Perform linear interpolation (Alpha Blending)
canvas[y_offset : y_offset + new_h, x_offset : x_offset + new_w] = \
    (bgr * alpha_3d) + (roi * (1.0 - alpha_3d))
\end{lstlisting}

\subsection{Augmentation Pipeline}
During every training iteration, every reference image is processed through a stochastic augmentation chain. To ensure the reproducibility of the trained models, the exact configuration of the implemented pipeline is detailed in Table~\ref{tab:augmentations}. The applied transformations are categorized by the physical phenomena they are designed to simulate:
\begin{itemize}
    \item Geometric Deformations: Real-world physical cards are rarely photographed perfectly orthogonal to the camera lens. Transformations such as affine scaling, translation, rotation, and perspective warping are applied to induce model invariance to camera pose, alignment errors, and variations in focal length.
    \item Optical and Environmental Simulation: Playing cards are frequently protected by glossy plastic sleeves, which introduce severe specular highlights and glare. To model this, operations such as \texttt{RandomSunFlare} and \texttt{GlassBlur} are utilized alongside standard motion blur and shadow generation. Furthermore, variations in brightness, contrast, and hue approximate diverse environmental lighting conditions (e.g., warm incandescent room lighting versus cool daylight).
    \item Sensor Noise and Occlusions: Consumer webcams and smartphone cameras frequently introduce compression artifacts and ISO noise under low-light conditions. These are modeled using \texttt{ImageCompression} and \texttt{ISONoise}. Finally, physical damage is simulated using \texttt{CoarseDropout} with random noise filling acting as a spatial regularizer that forces the network to learn distributed feature representations rather than relying on a single localized focal point.
\end{itemize}

\section{Theoretical Framework}

\subsection{Metric Learning Approach}
Standard image classification models typically employ the Softmax loss function to predict a probability distribution over a fixed set of classes. While effective for closed-set problems (e.g., ImageNet classification), this approach is fundamentally unsuitable for \ac{TCG} identification due to specific domain constraints:
\begin{itemize}
    \item High Cardinality and Open-Set Nature: \ac{MTG} contains over 100,000 unique card faces. Training a standard classifier with a final fully connected layer of size $N \approx 10^5$ is computationally expensive. More importantly, the domain is inherently open-set; new expansions are released frequently, rendering a fixed-size classifcation layer obsolete without constant retraining. % which is crucial for robust embedding retrieval
    \item Intra-Class Variance vs. Inter-Class Similarity: The model must be invariant to non-semantic variations such as foiling effects, card condition, or slight lightning changes. It must distinguish between similar but distinct cards (e.g., reprints with different set symbols or borders). Standard Softmax loss maximizes inter-class separability but lacks explicit constraints to enforce intra-class compactness.
\end{itemize}
To address these challenges, a \ac{DML} approach was adopted. Instead of classifying images directly, the goal is to learn a mapping function $f(\mathbf{x})$ that embeds input images into a low-dimensional Euclidean space (typically constrained to a hypersphere). In this space, the distance between embeddings corresponds directly to semantic similarity, allowing for card identification via Nearest Neighbor Search rather than direct classification.

\subsection{ArcFace: Additive Angular Margin Loss}\label{sec:arcface}
ArcFace (Additive Angular Margin Loss)~\cite{Deng_2022} was employed, a state-of-the-art objective function originally designed for large-scale facial recognition. The hypothesis is that \ac{TCG} recognition shares key characteristics with facial recognition: it requires subtle distinctions between thousands of classes based on fine-grained visual cues.

\subsection{Geometric Interpretation}
To understand ArcFace mechanics, it is best to start by analyzing the standard operations of the final fully connected layer of the classification network.

Let $\mathbf{x}_i \in \mathbb{R}^d$ denote the deep feature vector (embedding) of the $i$-th sample in the current batch, where $d$ is the embedding dimension ($d=512$ in our architecture).
Furthermore, let $\mathbf{W} \in \mathbb{R}^{d \times C}$ be the weight matrix of the final layer, where $C$ is the total number of classes (unique cards in our case).

The matrix $\mathbf{W}$ can be decomposed into $C$ column vectors $\mathbf{W} = \begin{bmatrix}\mathbf{W}_1, \mathbf{W}_2, \dots, \mathbf{W}_C\end{bmatrix}$, where each vector $\mathbf{W}_j \in \mathbb{R}^d$ acts as the learned \textit{class center} for the $j$-th class. In a standard linear layer (with bias set to zero), the activation (logit) for the $j$-th class given input $\mathbf{x}_i$ is
\begin{equation}
    \text{logit}_j = \mathbf{W}_j^\intercal \mathbf{x}_i.
\end{equation}
Geometrically, this dot product relates to the angle $\theta_{j,i}$ between the feature vector and the class center:
\begin{equation}
   \mathbf{W}_j^\intercal \mathbf{x}_i = \|\mathbf{W}_j\| \|\mathbf{x}_i\| \cos \theta_{j,i}.
\end{equation}
In traditional Softmax loss, the magnitudes $\|\mathbf{W}_j\|$ and $\|\mathbf{x}_i\|$ influence the output probability, which does not necessarily contribute to class separability. ArcFace addresses this by strictly enforcing normalization. By fixing $\|\mathbf{W}_j\| = 1$ and $\|\mathbf{x}_i\| = 1$ via $L_2$ normalization, the prediction depends \textit{solely} on the angle $\theta_{j,i}$:
\begin{equation}
    \mathbf{W}_j^\intercal \mathbf{x}_i = \cos \theta_{j,i}.
\end{equation}
This projects the learned features onto a hypersphere, where minimizing the loss equates to minimizing the geodesic distance (angle) between the sample $\mathbf{x}_i$ and its corresponding ground-truth class center $\mathbf{W}_{y_i}$. % TODO: je potřeba angle vůbec zmiňovat?

To enforce a margin between classes, ArcFace adds an angular penalty $m$ directly to the angle $\theta_{y_i, i}$ of the ground-truth class. Additionally, since the cosine range $[-1, 1]$ is insufficient for the Softmax function to produce a sharp probability distribution (leading to slow convergence), the logits are scaled by a fixed factor $s$.
\input{./fig/tikz/arcface.tex}
The final ArcFace loss for a sample $\mathbf{x}_i$ with label $y_i$ is defined as:
\begin{equation}
    L_{\text{ArcFace}} = -\log \frac{e^{s \cdot \cos(\theta_{y_i, i} + m)}}{e^{s \cdot \cos(\theta_{y_i, i} + m)} + \sum_{j \neq y_i} e^{s \cdot \cos(\theta_{j, i})}}
\end{equation}
By minimizing this loss, the network learns to map images of the same card to tightly clustered points on the hypersphere, while pushing different cards apart. Crucially, during the inference phase, the weight matrix $\mathbf{W}$ and the margin $m$ are discarded, and the recognition is performed solely by computing the cosine similarity between the query embedding $f(\mathbf{x})$ and the database of card embeddings.

The complete end-to-end inference pipeline, illustrated in Figure~\ref{fig:pipeline_flow}, consists of three high-level operational blocks: image preprocessing, feature extraction, and similarity matching.

However, the core neural network (\texttt{MTGReconModel}) utilized for the feature extraction is trained using a metric learning paradigm. Its underlying architecture is composed of three distinct modules designed to map raw pixel data into discriminative 512-dimensional Euclidean space.

\subsection{Backbone Network}
The foundational feature extractor is a ResNet-50~\cite{He2015} architecture, instantiated via the \texttt{timm} library. To adapt the network for metric learning, the final classification fully connected layer is omitted. The backbone is initialized with weights pre-trained on the ImageNet dataset to leverage learned low-level visual features such as edge detection and basic geometric pattern recognition. The network processes standardized input tensors of size $512 \times 512 \times 3$. To prevent catastrophic forgetting of the pre-trained weights while adapting to the \ac{TCG} domain, the backbone parameters are optimized using a learning rate ten times smaller than that of the newly initialized custom network heads.

\subsection{Embedding Head}\label{sub:embedhead}
The 2048-dimensional feature maps extracted by the ResNet-50 backbone are projected into a compact embedding space via a custom architectural neck. This embedding head consists of the following sequential operations:
\begin{enumerate}
    \item 1D Batch Normalization: Stabilizes the high-dimensional output from the backbone. During distributed training across multiple GPUs, this is converted to Synchronized Batch Normalization to ensure accurate batch statistic computation.
    \item Dropout: A dropout layer with a probability of $p=0.4$ is applied to regularize the network and mitigate overfitting, a common failure mode in fine-grained open-set classification.
    \item Linear Projection: A fully connected layer without bias parameters reduces the dimensionality from 2048 to the target embedding size of $d=512$.
    \item 1D Batch Normalization: A final normalization step applied immediately before the features are projected onto the hypersphere.
\end{enumerate}

\subsection{ArcFace Head}
During the training phase, the 512-dimensional embeddings are passed to the \texttt{ArcFaceHead} module, which implements the additive angular margin penalty described in \refsec{sec:arcface}. The head maintains a learnable weight matrix initialized via Xavier uniform initialization.

The calculated cosine similarities are limited to the interval $[-1.0 + 10^{-7}, \, 1.0 - 10^{-7}]$ to ensure numerical stability during backpropagation and to prevent undefined behavior of the $\arccos$ function.

It is important to note that this module is used exclusively during training. After deployment for inference, the \texttt{ArcFaceHead} weights are completely removed from the model's state dictionary. The pipeline then performs recognition by calculating the raw cosine similarity between the $L_2$-normalized 512-dimensional vectors produced by the \hyperref[sub:embedhead]{Embedding Head}.

\subsection{Dimensional Collapse}
During early training iterations, it was observed that the network would occasionally degenerate, producing an output vector approaching zero for all input images. In this state, the network effectively learns to ignore the input data, mapping all samples to a single point in the embedding space to trivially satisfy the intra-class distance minimization without learning meaningful inter-class separability.

To prevent this catastrophic failure and stabilize the training dynamics, several structural and optimization constraints were integrated into the pipeline:
\begin{enumerate}[(a)]
    \item Differential Learning Rates: The ResNet-50 backbone is initialized with ImageNet pre-trained weights, while the embedding and ArcFace heads are randomly initialized. Using a uniform learning rate across the entire network risks destroying the pre-trained low-level feature detectors. Therefore, the backbone parameters are optimized with a learning rate an order of magnitude smaller ($\eta = 10^{-5}$) than the network heads ($\eta = 10^{-4}$).
    \item Strict $L_2$ Normalization: By enforcing $L_2$ normalization on both the feature embeddings $\mathbf{x}_i$ and the class weights $\mathbf{W}_j$ prior to calculating the cosine similarity, the magnitudes of the vectors are factored out of the loss calculation. This mathematically prevents the network from minimizing the loss by shrinking the embedding magnitudes to zero.
\end{enumerate}
